Log:
- Magnus3: Social hierarchy + ecology system implementation
  - Added Community and Family entity types to the ECS
    - `CommunityIdentity` (seed, name, region, slot), `FoodStores` (grain)
    - `FamilyIdentity` (seed, surname, slot)
    - Hierarchy: Region -> Community (3) -> Family (3) -> Person (3-5), all child_of
  - New deterministic seed functions: `community_seed()`, `family_seed()`, `ecology_seed()`
  - Created CommunityGenerator, FamilyGenerator, WorldGenerator
    - Community names: syllabic root + place suffix (e.g. "Gurmholm", "Waemvale", "Frandale")
    - Families share surnames across members
    - WorldGenerator orchestrates full hierarchy from a single world seed
    - ~36 people per region (3 communities x 3 families x ~4 members)
  - Ecology Deity entity: singleton that drives all agricultural events
    - `EcologyState` component with per-region `RegionEcology` (soil fertility, rainfall, drought)
    - First Flecs system: `EcologySystem` fires at season boundaries (every 23,040 ticks)
    - Seasonal logic: Spring (weather), Summer (drought risk 10%), Autumn (harvest), Winter (consumption + famine check)
    - Food math: base 150 grain/community/year, modified by fertility x rainfall x variance
    - Consumption: 3 grain/person/season, ~12 people/community = ~144/year -> small surplus in good years
    - Drought reduces rainfall to 0.3, lasts 1-3 seasons -> harvests drop to ~40-50 -> famine
  - CLI inspect commands: complete game state visibility
    - `inspect world` - overview with communities, food stores, event counts
    - `inspect community <slot>` - families, members, events targeting this community
    - `inspect family <c> <f>` - members with personality traits and health
    - `inspect ecology` - deity state per region (fertility, rainfall, drought)
    - `inspect events` - chronological event log with details
    - All use `--seed` and `--tick` to regenerate deterministic world state
  - Flecs gotcha: empty tag structs can't use `.set<>()`, must use `.add<>()` or `.with<>()`
  - Flecs gotcha: `.each()` lambda can't take reference to empty tag component, use `.with<Tag>()` filter instead
  - Verified: determinism holds, 5-year run shows natural drought/recovery cycles
